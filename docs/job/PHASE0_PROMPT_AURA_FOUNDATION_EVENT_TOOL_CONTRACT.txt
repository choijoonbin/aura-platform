[PHASE 0 / Aura Prompt] 표준 Agentic 실행 구조 정비 — 이벤트 스키마/도구 계약/상태 동기화 규칙
(목표: Phase A에서 ‘보이는 Agentic’, Phase B에서 ‘탐지 배치’가 들어와도 Aura가 흔들리지 않는 기본 계약 확정)

## 0) 원칙
- 가정 금지(코드/도구/엔드포인트 기준).
- HITL 우선: propose→approve/reject→resume 기본 흐름 고정.
- 이벤트는 타입+payload 구조화(텍스트 로그 금지).
- trace_id/case_id/tenant_id/user_id를 모든 이벤트/툴 호출에 포함.

## 2) 구현 범위(In/Out)
### In
- Finance Agent 기준으로 이벤트 타입/스키마 확정
- tool 호출 표준화(입력 caseId → 관련 데이터 조회 → evidence_refs 생성)
- 실패/타임아웃 시 error 이벤트로 종료(침묵 금지)

### Out
- 신규 에이전트 대폭 확장
- 자동 실행(무승인) 조치

---

## 3) Definition of Done
- 이벤트 스키마/도구 계약/상태 동기화 문서가 실제 코드 기준으로 완성
- 샘플 케이스 1개를 넣었을 때(이미 존재한다고 가정) 이벤트 흐름이 재현 가능
